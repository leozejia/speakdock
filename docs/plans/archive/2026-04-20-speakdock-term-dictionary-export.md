# SpeakDock 阶段归档

## 1. 阶段主题

- 阶段：P1 `AI 语音输入法`
- focus：补齐 `TermDictionary` 导出闭环，让"可撤回 / 可删除 / 可导出"这条定义在实现上完整成立
- 状态：`Completed`

## 2. 为什么当时做

上一轮已经把诊断入口和 `HotPathCoordinator` 边界收口，但还有一个更直接的文档缺口没有补齐：

- 架构已经明确写了 `TermDictionary` 条目必须"可撤回、可删除、可导出"
- 实现里已经有添加、删除、忽略和本地学习
- 但还没有导出入口，定义和实现之间存在真实落差

这类问题比追新能力更值得先做，方向没有歧义，架构已经给出结论。

## 3. 本轮实际完成

- `TermDictionaryStore` 已补导出能力，并有回归测试
- `Settings -> Dictionary` 已接入显式导出入口
- 人工验收文档已同步，"可导出"已进入验证口径

## 4. 这轮锁住的边界

- 不做词典导入、云同步、多端同步
- 不把词典扩成知识库或 `StyleProfile`
- 不为导出补新的后台服务或数据库层
- 不重新改写当前词级学习规则

## 5. 验证结果

- `make test` -> pass
- 导出能力已有自动化测试，不只是手工点通
- 导出不改变现有词典学习和热路径行为

## 6. 结束判断

这一轮已经满足完成定义：

- `TermDictionary` 现在完整支持"可撤回 / 可删除 / 可导出"
- `CURRENT / review / manual test` 对"词典可导出"口径一致
- 导出能力有自动化回归覆盖
